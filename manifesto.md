# Manifesto

## Purpose

Chapeau Linux is an independent Linux distribution developed to provide a dependable operating system through a validated rolling-release model.

The project is founded on the belief that access to current software should not require compromising reliability, stability, or predictability. Chapeau Linux exists to make Linux more accessible by delivering an operating system that individuals, businesses, governments, and other organizations can adopt with confidence.

## Release Model

Chapeau Linux is developed as a validated rolling-release distribution.

Software is delivered continuously through a structured validation pipeline before reaching the stable repository. Automated testing, package validation, integration testing, staged deployment, and engineering review are used to identify regressions, resolve defects, and improve compatibility before updates are released.

The objective of this release model is to provide current software while maintaining a stable, dependable, and predictable operating system.

## Principles

The development of Chapeau Linux is guided by the following principles:

- Reliability is earned through disciplined engineering.
- Stability should not prevent access to current software.
- Changes should improve the operating system without introducing unnecessary risk.
- Development should be open, transparent, and well documented.
- Technical decisions should prioritize the long-term interests of the project and its users.

These principles define the foundation upon which Chapeau Linux is developed.
